# 安全策略

## 报告安全问题

如果你发现安全漏洞，请负责任地披露：

1. **不要**公开披露漏洞
2. 通过 [GitHub Issue](https://github.com/Kumagt/price-monitor/issues) 或邮件联系
3. 包含详细信息：
   - 漏洞描述
   - 复现步骤
   - 潜在影响
   - 修复建议（如有）

我们会在 3 个工作日内回复。

---

## 安全实践

### 数据安全

- ✅ 所有数据本地存储（SQLite）
- ✅ 不上传用户数据到第三方
- ✅ 不收集个人信息
- ✅ 开源代码可审查

### API 使用

- ✅ 邀请码可通过环境变量配置，不硬编码
- ✅ REST API 支持 Bearer Token 认证
- ✅ 遵守 API 使用限制
- ✅ 缓存减少请求频率

### 依赖管理

- ✅ 最小化依赖（仅 aiohttp）
- ✅ 使用 uv 锁定依赖版本
- ✅ 零外部框架依赖（Web UI 内嵌）

---

## 已知限制

### 价格准确性

- 实际价格以电商平台结算页为准
- 优惠券可能有时效性
- 促销活动可能突然变化

### API 限制

- 频繁请求可能触发限流
- 建议检查间隔不低于 30 分钟
- 某些商品可能无法获取详情
